A10:2025 Mishandling of Exceptional Conditions

Unhandled exceptions expose internal paths, config, and DB structure to attackers.

Challenges
  1. Stack trace exposure: Use /a10/calculate-bad — divide by zero (a=5, b=0). Find the absolute file path in the traceback.
  2. Path traversal via error: Use /a10/read-bad?file=../../etc/passwd. The stack trace reveals the real filesystem layout.
  3. SQL error leakage: Use /a10/query-bad?table=nonexistent. The error reveals internal table names and DB structure.
  4. Compare with fixed: Try the same attacks on /calculate-good, /read-good — observe generic errors only.
