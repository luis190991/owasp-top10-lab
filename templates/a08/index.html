A08:2025 Software & Data Integrity Failures

Code and data without integrity verification — insecure deserialization, unsigned updates.

Challenges
  1. Pickle RCE: The /a08/deserialize endpoint deserializes base64-encoded pickle data. Generate a malicious payload using /a08/generate-payload and send it to achieve code execution.
  2. Unsigned Update: Observe how /a08/update applies a version bump with no hash or signature verification.
Generate Payload Deserialize Unsigned Update Solution
How pickle RCE works: Python's pickle module can serialize arbitrary objects, including ones whose __reduce__ method runs OS commands when deserialized. Never deserialize untrusted pickle data.
