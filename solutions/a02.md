# A02 – Mala Configuración de Seguridad: Soluciones

## Reto 1 – Configuración Expuesta
```
GET /a02/config
```
Retorna JSON con el `SECRET_KEY`, `DEBUG=True` y todos los detalles internos de Flask.

**Corrección:** Elimina o restringe este endpoint por completo. Nunca expongas la configuración de la app vía HTTP.

## Reto 2 – Archivo de Respaldo
```
GET /a02/backup.cfg
```
Retorna credenciales en texto plano incluyendo la llave live de Stripe y el secreto de AWS.

**Corrección:** Nunca guardes ni sirvas archivos de configuración o credenciales. Usa variables de entorno y gestores de secretos.

## Reto 3 – Credenciales por Defecto
Navega a `/a02/login` e ingresa `admin` / `admin` → "LOGIN SUCCESS".

**Corrección:** Fuerza el cambio de contraseña en el primer despliegue. Usa contraseñas por defecto fuertes y generadas aleatoriamente.

## Reto 4 – Cabeceras Verbosas
```bash
curl -I http://localhost:5000/a02/
```
Las cabeceras de respuesta exponen:
- `X-Powered-By: Flask/3.0.3 Python/3.11 Werkzeug/3.0.3`
- `X-Debug-Mode: True`
- `Server: Apache/2.4.50 (Ubuntu)`

**Corrección:** Elimina las cabeceras que revelan tecnología. Agrega cabeceras de seguridad:
```python
response.headers['X-Content-Type-Options'] = 'nosniff'
response.headers['X-Frame-Options'] = 'DENY'
```

## Reto 5 – API Interna
```
GET /a02/api/data
```
Retorna nombres de usuarios internos, versión del servidor y cadena de conexión a la BD.

**Corrección:** Agrega autenticación. Nunca expongas la topología interna a través de una API pública.
