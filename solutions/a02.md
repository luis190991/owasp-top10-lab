# A02 – Security Misconfiguration: Solutions

## Challenge 1 – Exposed Config
```
GET /a02/config
```
Returns JSON with `SECRET_KEY`, `DEBUG=True`, and all Flask internals.

**Fix:** Remove/restrict this endpoint entirely. Never expose app config via HTTP.

## Challenge 2 – Backup File
```
GET /a02/backup.cfg
```
Returns plaintext credentials including Stripe live key and AWS secret.

**Fix:** Never commit or serve config/credential files. Use environment variables + secrets managers.

## Challenge 3 – Default Credentials
Navigate to `/a02/login` and enter `admin` / `admin` → "LOGIN SUCCESS".

**Fix:** Enforce password change on first deploy. Use strong randomly generated defaults.

## Challenge 4 – Verbose Headers
```bash
curl -I http://localhost:5000/a02/
```
Response headers expose:
- `X-Powered-By: Flask/3.0.3 Python/3.11 Werkzeug/3.0.3`
- `X-Debug-Mode: True`
- `Server: Apache/2.4.50 (Ubuntu)`

**Fix:** Remove technology-revealing headers. Add security headers:
```python
response.headers['X-Content-Type-Options'] = 'nosniff'
response.headers['X-Frame-Options'] = 'DENY'
```

## Challenge 5 – Internal API
```
GET /a02/api/data
```
Returns internal usernames, server version, and DB connection string.

**Fix:** Add authentication. Never expose internal topology via public API.
