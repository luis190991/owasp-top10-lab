# A04 – Fallos Criptográficos: Soluciones

## Reto 1 – Romper los Hashes MD5
Desde `/a04/hashes`, los hashes son:
- `alice` → `482c811da5d5b4bc6d497ffa98491e38` = `password123`
- `bob`   → `3a85339b80d53a6c4adea87660b66bb4` = `bob123`
- `carol` → `e10adc3949ba59abbe56e057f20f883e` = `123456`

Crackéalos con CrackStation o hashcat:
```bash
echo "482c811da5d5b4bc6d497ffa98491e38" > hashes.txt
hashcat -a 0 -m 0 hashes.txt rockyou.txt
```

**Corrección:** Usa bcrypt con sal por usuario:
```python
import bcrypt
hashed = bcrypt.hashpw(password.encode(), bcrypt.gensalt())
bcrypt.checkpw(candidato.encode(), hashed)
```

## Reto 2 – Base64 No Es Cifrado
En `/a04/store`, codifica `password=Admin123`:
```
cGFzc3dvcmQ9QWRtaW4xMjM=
```
Decodifícalo al instante:
```bash
echo "cGFzc3dvcmQ9QWRtaW4xMjM=" | base64 -d
```
**Corrección:** Usa AES-GCM o ChaCha20-Poly1305 con una llave generada aleatoriamente y almacenada en un gestor de secretos.

## Reto 3 – Token de Sesión Predecible
Después de iniciar sesión como `alice`, el token es:
```
MD5("alice:static_salt_2024")
```
Calcula el token de `bob` sin iniciar sesión:
```bash
python3 -c "import hashlib; print(hashlib.md5(b'bob:static_salt_2024').hexdigest())"
```

**Corrección:** Usa `secrets.token_hex(32)` para los tokens de sesión. Almacénalos en el servidor, nunca los derives de datos del usuario.
