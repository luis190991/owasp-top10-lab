# A04 – Cryptographic Failures: Solutions

## Challenge 1 – Crack MD5 Hashes
From `/a04/hashes`, the hashes are:
- `alice` → `482c811da5d5b4bc6d497ffa98491e38` = `password123`
- `bob`   → `3a85339b80d53a6c4adea87660b66bb4` = `bob123`
- `carol` → `e10adc3949ba59abbe56e057f20f883e` = `123456`

Crack using CrackStation or hashcat:
```bash
echo "482c811da5d5b4bc6d497ffa98491e38" > hashes.txt
hashcat -a 0 -m 0 hashes.txt rockyou.txt
```

**Fix:** Use bcrypt with per-user salt:
```python
import bcrypt
hashed = bcrypt.hashpw(password.encode(), bcrypt.gensalt())
bcrypt.checkpw(candidate.encode(), hashed)
```

## Challenge 2 – Base64 Is Not Encryption
At `/a04/store`, encode `password=Admin123`:
```
cGFzc3dvcmQ9QWRtaW4xMjM=
```
Decode instantly:
```bash
echo "cGFzc3dvcmQ9QWRtaW4xMjM=" | base64 -d
```
**Fix:** Use AES-GCM or ChaCha20-Poly1305 with a randomly generated key stored in a secrets manager.

## Challenge 3 – Predictable Session Token
After logging in as `alice`, the token is:
```
MD5("alice:static_salt_2024")
```
Compute for `bob` without logging in:
```bash
python3 -c "import hashlib; print(hashlib.md5(b'bob:static_salt_2024').hexdigest())"
```

**Fix:** Use `secrets.token_hex(32)` for session tokens. Store server-side, never derive from user data.
