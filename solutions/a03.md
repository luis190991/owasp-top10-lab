# A03 – Supply Chain Failures: Solutions

## Challenge 1 – Dependency Audit
```bash
pip-audit
# or via the web UI at /a03/audit
```
Expected findings:
- `Pillow 9.3.0` → CVE-2023-44271 (DoS), CVE-2023-50447 (RCE)
- `PyYAML 5.3.1` → CVE-2020-14343 (arbitrary code execution)

**Fix:** Pin to latest patched versions:
```
Pillow>=10.3.0
PyYAML>=6.0.1
```
Add `pip-audit` to CI pipeline to catch new CVEs automatically.

## Challenge 2 – Unsafe YAML
Submit this payload at `/a03/parse-config`:
```yaml
!!python/object/apply:os.system
- "touch /tmp/yaml_pwned"
```
Then check: `ls /tmp/yaml_pwned`

**Fix:** Use `yaml.safe_load()` instead of `yaml.load()`:
```python
# VULNERABLE
result = yaml.load(config_text, Loader=yaml.Loader)

# FIXED
result = yaml.safe_load(config_text)
```
`safe_load` only handles basic types and raises an error on Python object tags.

## Challenge 3 – Unsigned Plugin
Navigate to `/a03/install-plugin` and install any plugin.
The "installation" proceeds with no sha256 verification.

**Fix:**
```python
import hashlib, requests

def install_verified_plugin(url, expected_sha256):
    data = requests.get(url).content
    actual = hashlib.sha256(data).hexdigest()
    if actual != expected_sha256:
        raise ValueError(f"Integrity check failed: {actual} != {expected_sha256}")
    # only now proceed with installation
```
