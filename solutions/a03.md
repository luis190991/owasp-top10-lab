# A03 – Fallos en la Cadena de Suministro: Soluciones

## Reto 1 – Auditoría de Dependencias
```bash
pip-audit
# o vía la interfaz web en /a03/audit
```
Hallazgos esperados:
- `Pillow 9.3.0` → CVE-2023-44271 (DoS), CVE-2023-50447 (RCE)
- `PyYAML 5.3.1` → CVE-2020-14343 (ejecución de código arbitrario)

**Corrección:** Actualiza a las últimas versiones con parches:
```
Pillow>=10.3.0
PyYAML>=6.0.1
```
Agrega `pip-audit` al pipeline de CI para detectar nuevos CVEs automáticamente.

## Reto 2 – YAML Inseguro
Envía este payload en `/a03/parse-config`:
```yaml
!!python/object/apply:os.system
- "touch /tmp/yaml_pwned"
```
Luego verifica: `ls /tmp/yaml_pwned`

**Corrección:** Usa `yaml.safe_load()` en lugar de `yaml.load()`:
```python
# VULNERABLE
result = yaml.load(config_text, Loader=yaml.Loader)

# CORREGIDO
result = yaml.safe_load(config_text)
```
`safe_load` solo maneja tipos básicos y lanza un error si encuentra etiquetas de objetos Python.

## Reto 3 – Plugin Sin Firma
Navega a `/a03/install-plugin` e instala cualquier plugin.
La "instalación" procede sin ninguna verificación de sha256.

**Corrección:**
```python
import hashlib, requests

def instalar_plugin_verificado(url, sha256_esperado):
    data = requests.get(url).content
    sha256_real = hashlib.sha256(data).hexdigest()
    if sha256_real != sha256_esperado:
        raise ValueError(f"Fallo de integridad: {sha256_real} != {sha256_esperado}")
    # solo ahora se procede con la instalación
```
