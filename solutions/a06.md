# A06 – Insecure Design: Solutions

## Challenge 1 – Predictable Reset Token
The token for `admin` is:
```bash
python3 -c "import hashlib; print(hashlib.md5(b'admincorp2024').hexdigest())"
# → e35b9dc90255e3b7bc5f2e8a21f16d7b (example)
```
Use this at `/a06/reset-password` to reset admin's password without access to their email.

**Fix:** Use `secrets.token_urlsafe(32)` — a 256-bit cryptographically random token.
Store it with a short TTL (15 min) in the DB. Never derive tokens from user data.

## Challenge 2 – Negative Quantity Cart
At `/a06/cart`, set quantity to `-10` for Laptop ($999.99):
- Subtotal = -10 × $999.99 = **-$9999.90**

The store now owes the attacker money.

**Fix:**
```python
quantity = int(request.form.get('quantity', 1))
if quantity <= 0:
    abort(400, "Quantity must be positive")
```

## Challenge 3 – Mass Assignment
Login as `alice`, then:
```bash
curl -X POST http://localhost:5000/a06/update-profile \
  -b "session=..." \
  -d "role=admin&balance=99999"
```
The backend executes `UPDATE a06_users SET role='admin' WHERE username='alice'`.

**Fix:**
```python
ALLOWED_UPDATE_FIELDS = {'email', 'password'}
for field, value in request.form.items():
    if field in ALLOWED_UPDATE_FIELDS:
        db.execute(f"UPDATE a06_users SET {field}=? WHERE username=?", ...)
```
