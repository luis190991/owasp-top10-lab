# A10 – Mishandling of Exceptional Conditions: Solutions

## Challenge 1 – Stack Trace via Division by Zero
At `/a10/calculate-bad`, enter `a=5`, `b=0`.
The response contains the full Python traceback including:
- Absolute file path: `/home/user/lab_dev_sec/app/labs/a10_exceptions.py`
- Line numbers
- Variable values
- Python version

**Fix:**
```python
try:
    result = int(a) / int(b)
except ZeroDivisionError:
    error = "Cannot divide by zero."
except ValueError:
    error = "Please enter valid integers."
except Exception:
    app.logger.exception("Unexpected error in calculate")
    error = "An unexpected error occurred."
```

## Challenge 2 – Path Traversal via Error
Navigate to:
```
/a10/read-bad?file=../../etc/passwd
```
The stack trace reveals the real filesystem path even on `FileNotFoundError`.
If `/etc/passwd` is readable, its contents are returned.

Compare with `/a10/read-good?file=../../etc/passwd` → "Access denied" only.

**Fix:** Always sanitize with `os.path.realpath()` and check the result starts
with the allowed base directory before opening.

## Challenge 3 – SQL Error Leakage
Navigate to:
```
/a10/query-bad?table=nonexistent_table
```
Error: `no such table: nonexistent_table` — confirms DB engine is SQLite,
reveals table names don't match.

Try `/a10/query-bad?table=a04_users` — returns all password hashes!

**Fix:**
- Whitelist allowed table names
- Return only generic errors: "Query failed. Please try again."
- Log the full SQL error server-side only

## General Pattern
```python
# BAD — exposes details
except Exception as e:
    return str(e), 500          # or traceback.format_exc()

# GOOD — generic to user, detailed to logs
except Exception:
    current_app.logger.exception("Internal error")
    return "An error occurred", 500
```
