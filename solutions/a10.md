# A10 – Manejo Incorrecto de Excepciones: Soluciones

## Reto 1 – Stack Trace por División Entre Cero
En `/a10/calculate-bad`, ingresa `a=5`, `b=0`.
La respuesta contiene el stack trace completo de Python incluyendo:
- Ruta absoluta del archivo: `/home/user/lab_dev_sec/app/labs/a10_exceptions.py`
- Números de línea
- Valores de variables
- Versión de Python

**Corrección:**
```python
try:
    result = int(a) / int(b)
except ZeroDivisionError:
    error = "No se puede dividir entre cero."
except ValueError:
    error = "Por favor ingresa números enteros válidos."
except Exception:
    app.logger.exception("Error inesperado en calculate")
    error = "Ocurrió un error inesperado."
```

## Reto 2 – Traversal de Ruta Vía Error

```

```

con el directorio base permitido antes de abrir cualquier archivo.

Navega a:
```

```
Error: `no such table: nonexistent_table` — confirma que el motor de BD es SQLite
y revela que los nombres de tabla no coinciden.

Prueba `/a10/query-bad?table=a04_users` — ¡retorna todos los hashes de contraseñas!

**Corrección:**
- Lista blanca de nombres de tabla permitidos

- Registrar el error SQL completo solo en el servidor

## Patrón General
```python
# MAL — expone detalles al usuario
except Exception as e:
    return str(e), 500          # o traceback.format_exc()

# BIEN — genérico al usuario, detallado en logs

```
